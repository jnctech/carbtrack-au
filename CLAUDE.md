# CLAUDE.md — CarbTrack AU

## Project Purpose
CarbTrack AU is a self-hosted API providing a validated Australian food carbohydrate database for Type 1 Diabetes management. `carbs_per_100g` is the hero metric driving schema priority, conflict detection, and UI.

## Tech Stack
**Python 3.12** | **FastAPI** | **SQLModel** | **SQLite** (Postgres-ready) | **Docker** | **ruff** | **pytest**

## AI Assistant Usage
| Context | Model | Rationale |

| All development (spec, implementation, review, CI) | Claude Opus 4.6 | Max plan — Opus end-to-end avoids schema drift from model handoffs |
| Runtime only: query construction + field mapping | Claude Sonnet (`claude-sonnet-4-5` via `SONNET_MODEL` env var) | Bounded to 4 endpoints, never in data path |

## Hard Constraints (Non-Negotiable)
1. **Data Rules**: All nutrition values stored strictly `per 100g`. Energy stored as `energy_kj` (kJ, not calories). Never use serving-adjusted values. Soft deletes only (`active=false`) — never `DELETE FROM foods`.
2. **Database Portability**: No Postgres-specific SQL or SQLite-specific types. JSON-carrying fields use `str` (Text), not `JSON` type. Migration is `DATABASE_URL` swap only.
3. **Runtime AI Policy**: Claude Sonnet restricted to query construction and field mapping only (4 endpoints). Zero AI in runtime data retrieval, conflict detection, or storage. Max tokens: 400 (query), 600 (mapping). Model read from `SONNET_MODEL` env var — never hardcode.
4. **Quality Gates**: Zero `ruff` errors, ≥80% test coverage, cognitive complexity ≤15, SonarCloud Grade A (project: `jnctech_carbtrack-au`, org: `jnctech`, target branch: `develop`). All GitHub Actions SHA-pinned.
5. **Secrets**: `ANTHROPIC_API_KEY` in `.env` only — never hardcode, never log. `raw_response_json` in `food_source_refs` — never log, never return in API responses. IPs as env vars — never hardcode.
6. **Commits**: Conventional commits with scope. AI-assisted commits: `Co-Authored-By: Claude Opus 4.6 (1M context) <noreply@anthropic.com>`.
